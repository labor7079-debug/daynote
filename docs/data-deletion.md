# DayNote 계정·데이터 삭제 요청

**시행일: 2026년 7월 9일 · 최종 수정: 2026년 7월 9일**
**게시자: 강태영 · 문의: labor7079@gmail.com**

이 페이지는 **DayNote**(이하 "앱") 이용자가 자신의 데이터를 삭제하는 방법을 안내합니다.

---

## 1. 핵심 요약

- 앱은 **오프라인 우선**으로 설계되어, 메모·할 일 등 모든 데이터는 기본적으로 **이용자의 기기 안에만** 저장됩니다.
- **개발자는 별도의 서버를 운영하지 않으며, 이용자를 위한 계정을 만들지 않습니다.** 따라서 개발자가 보관하는 이용자 데이터는 없습니다.
- 데이터 전송이 일어나는 **선택 기능**(구글 캘린더 동기화, AI, 클라우드 동기화)은 모두 **이용자 본인 계정**으로 연결되며, 해당 데이터는 그 서비스에서 본인이 직접 삭제할 수 있습니다.

---

## 2. 기기 안의 데이터 삭제

- **개별 삭제**: 앱에서 각 메모·할 일을 직접 삭제할 수 있습니다.
- **전체 삭제**: 기기에서 앱을 **제거(삭제)**하면 앱이 저장한 모든 로컬 데이터(메모·할 일·AI 결과·설정·저장된 토큰/키)가 함께 삭제됩니다.
- (Android) 설정 → 애플리케이션 → DayNote → **저장공간 → 데이터 삭제**로도 초기화할 수 있습니다.

---

## 3. 외부로 전송된 데이터 삭제 (선택 기능 사용 시)

아래 기능을 켰던 경우, 각 서비스에서 **이용자 본인 계정으로** 삭제합니다.

- **구글 캘린더 동기화**: 앱이 만든 일정은 이용자 본인의 구글 캘린더에 있습니다 → [Google 캘린더](https://calendar.google.com)에서 해당 일정을 삭제하고, 앱 설정에서 동기화를 끄거나 로그아웃하면 기기에 저장된 접근 토큰이 제거됩니다.
- **클라우드 동기화(Supabase)**: 이용자 본인이 지정한 Supabase 프로젝트에 저장됩니다 → 본인의 Supabase 프로젝트에서 데이터를 삭제하고, 앱에서 로그아웃하면 로그인 토큰이 제거됩니다.
- **AI(OpenAI)**: 전송된 텍스트의 처리·보관은 OpenAI 정책을 따릅니다 → [OpenAI 개인정보처리방침](https://openai.com/policies/privacy-policy) 및 본인 OpenAI 계정에서 관리합니다. 앱에 저장된 AI 결과는 앱에서 개별 삭제하거나 앱을 제거하면 삭제됩니다.
