# DayNote 개인정보 처리방침

**시행일: 2026년 7월 1일 · 최종 수정: 2026년 7월 1일**
**게시자: 강태영 · 문의: labor7079@gmail.com**

본 방침은 **DayNote**(이하 "앱")가 이용자의 정보를 어떻게 다루는지 설명합니다. 앱은 **오프라인 우선(offline-first)**으로 설계되어, 모든 메모·할 일은 기본적으로 **이용자의 기기 안에만** 저장됩니다.

---

## 1. 핵심 요약

- 앱은 **이용자 데이터를 개발자에게 전송하지 않습니다.** 개발자는 별도의 서버를 운영하지 않으며, 이용자의 메모·할 일·계정 정보를 수집하거나 보관하지 않습니다.
- 이용자가 직접 켠 **선택 기능**(구글 캘린더 동기화, AI 기능, 클라우드 동기화)에 한해, 데이터가 **이용자 본인이 지정한 외부 서비스**로 전송됩니다.
- 광고, 분석(애널리틱스), 추적(트래킹) 도구를 **사용하지 않습니다.**

---

## 2. 앱이 다루는 정보

이용자가 앱에서 직접 작성·입력하는 정보입니다.

- **메모**: 제목, 본문(마크다운), 고정 여부, 날짜
- **할 일**: 내용, 완료 여부, 마감 날짜·시간
- **AI 결과**: AI 기능 사용 시 생성된 결과 텍스트
- **앱 설정**: 테마, 각 동기화 기능의 사용 여부 등
- **인증·연결 정보**: (선택 기능 사용 시) OpenAI API 키, 클라우드 접속 정보 및 로그인 토큰, 구글 캘린더 접근 토큰

위 정보는 기본적으로 **기기 내부 저장소(로컬 데이터베이스)**에 저장됩니다. 인증·연결 정보 등 민감한 값은 **기기의 보안 저장소(암호화)**에 보관됩니다.

---

## 3. 정보가 외부로 전송되는 경우 (모두 선택 기능)

아래 기능은 **기본적으로 꺼져 있으며**, 이용자가 설정에서 직접 켜고 본인 계정/키를 입력한 경우에만 동작합니다.

### 3.1 구글 캘린더 동기화
- 이용자가 본인 **구글 계정**으로 로그인·동의하면, 앱이 만든 메모·할 일의 **제목·날짜·시간**이 **이용자 본인의 구글 캘린더**에 일정으로 생성/수정/삭제됩니다.
- 접근 범위는 캘린더 일정(`calendar.events`)에 한정됩니다. 접근 토큰은 기기 보안 저장소에 저장됩니다.
- 전송 대상: **Google LLC** — [Google 개인정보처리방침](https://policies.google.com/privacy)

### 3.2 AI 기능 (OpenAI)
- 이용자가 본인 **OpenAI API 키**를 입력한 경우에만 사용할 수 있습니다.
- 요약·확장·교정·질문·제목 생성 등을 실행하면, **해당 메모의 텍스트**가 처리를 위해 OpenAI로 전송되고 결과가 앱에 저장됩니다.
- 전송 대상: **OpenAI** — [OpenAI 개인정보처리방침](https://openai.com/policies/privacy-policy)

### 3.3 클라우드 동기화 (Supabase)
- 이용자가 본인 **Supabase 프로젝트 접속 정보(URL·키)**와 **로그인 계정(이메일·비밀번호)**을 입력한 경우에만 동작합니다.
- 여러 기기의 데이터를 맞추기 위해, **메모·할 일**이 **이용자 본인이 지정한 Supabase 프로젝트**에 저장·동기화됩니다.
- 전송 대상: 이용자가 지정한 **Supabase** 프로젝트 — [Supabase 개인정보처리방침](https://supabase.com/privacy)

> 위 세 서비스는 모두 **이용자 본인의 계정/키**로 연결됩니다. 데이터가 전송된 이후에는 각 서비스의 정책과 이용자의 계정 설정에 따라 관리됩니다.

## 4. 정보 보관 및 삭제

- **로컬 데이터**: 기기에 저장되며, 앱을 삭제하면 함께 삭제됩니다. 앱 내에서 개별 메모·할 일을 삭제할 수도 있습니다.
- **외부로 전송된 데이터**: 구글 캘린더의 일정, Supabase에 저장된 데이터, OpenAI로 전송된 내용은 각 서비스에서 **이용자가 직접 관리·삭제**할 수 있습니다.
- **인증 정보**: 각 동기화 기능을 끄거나 로그아웃하면 기기에 저장된 토큰이 제거됩니다.
- 데이터 삭제 방법은 별도 안내 페이지를 참고하세요: [계정·데이터 삭제 요청](data-deletion.html)
