# Play 데이터 보안(Data safety) 양식 — 작성 가이드

> Play Console → **앱 콘텐츠 → 데이터 보안**에서 묻는 항목에 대한 **권장 답안**입니다. 이 앱의 실제 동작 기준이며, [개인정보 처리방침](privacy-policy.md)과 내용이 일치합니다.
> ⚠️ 최종 판단은 게시자 책임입니다. Google의 정의상 **"수집=기기 밖으로 전송", "공유=제3자에게 전송"**이므로, 사용자가 켠 선택 기능으로 데이터가 외부(구글·OpenAI·Supabase)로 나가는 이 앱은 **"데이터를 전송한다"고 정직하게 신고**하는 것이 안전합니다. ("아무것도 수집 안 함"으로 신고하면 안 됨 — 기능을 켜면 실제로 전송되므로.)

## 0. 큰 원칙
- **개발자(나)는 어떤 데이터도 내 서버로 수집하지 않는다.** 별도 서버가 없다.
- 다만 사용자가 켠 기능에 한해 **사용자 본인 계정의 외부 서비스**로 데이터가 전송된다 → 이건 신고 대상.
- **광고·분석·광고 ID 없음.**

## 1. 첫 화면 질문
| 질문 | 답 |
|---|---|
| 앱이 필수 사용자 데이터 유형을 수집/공유합니까? | **예** (아래 유형 참조) |
| 전송 중 데이터가 암호화됩니까? | **예** (모든 외부 통신 HTTPS/TLS) |
| 사용자가 데이터 삭제를 요청할 방법이 있습니까? | **예** (앱 삭제 시 로컬 삭제 · 각 서비스에서 본인 삭제 · 문의 이메일) |

## 2. 수집·공유하는 데이터 유형
> 각 유형: **수집/공유 여부**, **목적=앱 기능(App functionality)**, **선택(Optional)**(기본 꺼짐), **임시처리 아님**(동기화는 저장됨).

| 데이터 유형(Play 분류) | 수집 | 공유(제3자) | 언제 | 대상 |
|---|---|---|---|---|
| **캘린더 → 캘린더 일정** | — | **예** | 구글 캘린더 동기화 ON | 구글(사용자 본인 캘린더) |
| **개인정보 → 이메일 주소** | **예** | **예** | 클라우드 동기화 로그인 시 | Supabase(사용자 본인 프로젝트) |
| **앱 활동/기타 → 사용자 생성 콘텐츠**(메모·할 일 텍스트) | **예** | **예** | AI 기능/클라우드 동기화 ON | OpenAI, Supabase(사용자 본인) |

- 모두 **목적: 앱 기능**, **선택(사용자가 켠 경우만)**, **계정 관리 목적의 이메일**은 "계정 관리"도 함께 체크 가능.

## 3. 수집/공유하지 **않는** 것 (전부 "아니오")
- 위치, 연락처, 전화번호, 사진·동영상, 음성, 금융·결제, 건강·피트니스, 검색·브라우징 기록
- **기기·광고 식별자(광고 ID)** — 광고·분석 SDK가 없으므로 없음
- 개발자 자체 분석/크래시 수집 없음

## 4. 보안 관행
- **전송 중 암호화: 예** (TLS)
- **삭제 요청 가능: 예** — 앱 삭제(로컬), 각 외부 서비스에서 본인 계정으로 삭제, 문의(labor7079@gmail.com)
- (선택) 독립 보안 검토 여부 등은 해당 없으면 비워둠

## 5. 광고 ID 권한
- 이 앱은 광고를 넣지 않았으므로 `AD_ID` 권한 없음 → "광고 또는 마케팅 목적 사용 안 함".
